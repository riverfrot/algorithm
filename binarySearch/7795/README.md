#문제 풀이
링크 
백준 7795문제 (링크 : https://www.acmicpc.net/problem/7795)

## 제한조건
시간제한 1초
메모리제한 128MB

## 시간 복잡도 분석 및 전략
단순 선형 탐색시
O(T*N*M) = 20000*20000 = 4억 테스트 케이스 까지 고려시 1초 제한초과

이분 탐색으로 구현시
O(T*MlogM*NlogM) second 집단 정렬(MlogM) 추후 첫번째ㅐ 집단을 기준으로 이분탐색하면
최대 20000인경우에도 300,000*T 연산으로 충분히 처리가능

### 해결방안
위에 대한 해결방안으로 선형 방식이아닌 한 집단을 정렬후 이분탐색으로 비교하는 방식의 해결방안이 필요
O(T*N*logM)이런형식이되어 최대 O(T*20000*log20000)이 되어 1초안에 실행이 가능


### 예외케이스
A의 최소값이 B의 최대값보다 큰 경우: 모든 B를 먹을 수 있음 (결과: N\times M).
A의 최대값이 B의 최소값보다 작은 경우: 아무것도 먹을 수 없음 (결과: 0).
두 배열의 원소가 모두 같은 경우: 자신보다 작은 것만 먹으므로 결과는 0.

### 고민했던 흔적들
초기 테스트 케이스 T가 있음 T는 제한 조건이 어느정도인지 문제에 나와있지않음... 
N,M 모두 1<=N,M <= 20,000 사이로 조건이 제한됨 
단순 선형 탐색으로는 O(20,000*20,000) 의 경우 4억*T 의 시간이 걸리기 때문에 시간 초과가 발생
