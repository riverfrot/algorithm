백준 14627문제 (링크 : https://www.acmicpc.net/problem/14627)

## 제한조건
시간제한 2초
메모리제한 256MB

## 시간 복잡도 분석 및 전략
단순 선형 탐색시
O(N*M) = 1,000,000*1,000,000 = 10억 시간초과
이분 탐색(매개변수 검색)으로 구현시
이분 탐색 사용 시: 매개변수 탐색시 O(NlogM)이 되어 N, M 모두 최악의 경우 1,000,000일때 600만번 연산만 하면되어 시간초과 이슈없음

### 해결방안
시간 복잡도 전략:
이분 탐색(매개변수 검색)으로 구현시

### 예외케이스
주의 해야 할점은 지만 하나의 파닭에는 하나 이상의 파가 들어가면 안 된다. 이 내용잉 지문에 들어가기 때문에
파의 길이 보다 길어지면 버림 처리를해서 남은 파의 길이르르 구해나가야함

### 고민했던 흔적들

위 내용은 파의 갯수 및 구해내야 하는 파닭의 갯수가 1,000,000 이므로 선형 탐색으로 10억의 케이스를 테스트 하기에 불가
O(N*M) = 시간 복잡도 초과 (자바에서는 1초에 1억정도를 기준 선)
O(NlogM) = 600만 정도여서 충분히 최대 케이스에서도 통과 가능
