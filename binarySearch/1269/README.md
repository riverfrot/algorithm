#문제 풀이
링크 
백준 1269문제 (링크 : https://www.acmicpc.net/problem/7795)

## 제한조건
시간제한 2초
메모리제한 256MB

## 시간 복잡도 분석 및 전략
단순 선형 탐색시
O(N*M) = 200,000*200,000 = 40억 시간초과

이분 탐색으로 구현시
O(MlogN*NlogM)인가 O(logN*logM) 확인필요
이분 탐색 사용 시: 한 배열을 정렬 O(NlogN), 다른 배열의 원소를 M개씩 이분 탐색 하였을때 O(MlogN)이 되므로
총 O((N+M)logN)이 되어 N, M 모두 최악의 경우 400,000일때 720만번 연산만 하면되어 시간초과 이슈없음

### 해결방안
시간 복잡도 전략:
HashMap을 이용한 O(N+M)로 400,000 풀이 선택.최대 40만 건의 데이터 처리가 필요하며, HashMap의 $O(1)$ 접근으로 2초 내에 넉넉히 통과 가능.

### 예외케이스
원소의 값은 최대 1억이지만, 개수는 20만 개이므로 int 배열이나 HashMap<Integer, Integer>로 충분히 수용 가능.
