백준 11053문제 (링크 : https://www.acmicpc.net/problem/11053)

## 제한조건
시간제한 1초
메모리제한 256MB

## 시간 복잡도 분석 및 전략
DP로도 O(N^2)으로 풀이가 가능하나 이분탐색으로 구현(LIS)
100만 -> 1만 정도로 시간복잡도 효율화

### 해결방안
LIS를 통해 기존값보다 같거나 작으면 왼쪽에 있는항목을교체 크다면 오른쪽 arrayList를 키우는 방식으로 구현

### 고민했던 흔적들
DP로 구현시 O(N^2) = 1,000*1,000 = 1000000인데 시간초과가 안될꺼같은데?
풀이방법은 이분탐색이긴함
기존 배열을 입력 받아서 새로운 배열을 만들고 여기서 기존 배열보다 작은 값이면 작은 갑을 기준으로 셋팅
값이 크면 오른쪽으로 셋팅

이분 탐색(매개변수 검색)으로 구현시
이분 탐색 사용 시: 매개변수 탐색시 O(NlogN)이 되어 10000정도로 해결가능?
