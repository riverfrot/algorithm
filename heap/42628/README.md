## 제한조건
operations는 길이가 1 이상 1,000,000 이하인 문자열 배열입니다.
operations의 원소는 큐가 수행할 연산을 나타냅니다.
원소는 “명령어 데이터” 형식으로 주어집니다.- 최댓값/최솟값을 삭제하는 연산에서 최댓값/최솟값이 둘 이상인 경우, 하나만 삭제합니다.
빈 큐에 데이터를 삭제하라는 연산이 주어질 경우, 해당 연산은 무시합니다.

## 시간 복잡도 분석 및 전략
O(NlogN)

### 해결방안
TreeMap 사용 (TreeSet은 중복 허용 X, 이중 우선순위큐는 시간 복잡도가 O(N))

### 고민했던 흔적들
이중우선순위 큐를 두개를만들어섯 해결하는 방법도있지만, 시간복잡도, 공간복잡도를 생각하면
TreeMap, TreeSet 자료구졸를 사용하느게 훨씬더 효율적이다.
두개의 힙을 구성하였을때는 
TreeMap, TreeSet, 이중큐 모두 add에는 O(logN)의 시간이 들어가나
pollMin, pollMax에는 이중큐에서는 O(N)의 시간복잡도가 소요되기 때문
TreeMap, TreeSet의 경우 균형 이진탐색트리로 모든노드가 정렬되어 있어
최소,최대값 탐색시 O(logN)의 시간복잡도를 가지게된다
