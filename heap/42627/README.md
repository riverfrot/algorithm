## 제한조건
≤ jobs의 길이 ≤ 500
jobs[i]는 i번 작업에 대한 정보이고 [s, l] 형태입니다.
s는 작업이 요청되는 시점이며 0 ≤ s ≤ 1,000입니다.
l은 작업의 소요시간이며 1 ≤ l ≤ 1,000입니다.

우선순위 작업의 소요시간이 짧은 것, 작업의 요청 시각이 빠른 것, 작업의 번호가 작은 것 순으로 우선순위가 높습니다.

## 시간 복잡도 분석 및 전략
O(NlogN)

### 고민했던 흔적들
디스크 컨트롤러는 작업시점에 따라서 현재 cpu가 바로 작업할 수 있는내용을 제공 할 수있어야함.
핵심은 지금 당장 할 수 있는 일 중에서 제일 빨리 끝낼 수 있는 것을 찾는것이를 위해
사전에 요청 시점을 기준으로 정렬을 먼저하고 작업이 가능한 시점중에서 작업시간이 짧은것을 우선순위 큐에 담음
초기 배열정렬에 O(NlogN), 우선순위 큐 삽입/삭제: 각 작업당 한 번씩 수행되므로 N*logN = O(NlogN)의 시간복잡도가 소요됨
