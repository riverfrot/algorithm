#문제 풀이
링크 
프로그래머스 H-Index (링크 : https://school.programmers.co.kr/learn/courses/30/lessons/42747)

## 제한조건
제한 사항
과학자가 발표한 논문의 수는 1편 이상 1,000편 이하입니다.
논문별 인용 횟수는 0회 이상 10,000회 이하입니다.

## 시간 복잡도 분석 및 전략
시간 복잡도 (정렬 이용) : O(NlogN)
공간 복잡도 : O(N)

### 해결방안
정렬과 복사를 사용

### 예외케이스

### 고민했던 흔적들
요지는 이거임 만약 10000이라는 숫자는 나올 수없음
왜냐하면 비교 조건에서 h값이 1000 이하인 숫자만 필요하니까
그러면 1000이하인 숫자를 기준으로 모두 비교해야하나? 이분탐ㅅ색? 
정답은 1<answer<1000 사이
이분탐색으로도 가능할꺼같네 가장 중요한 아 그러면되네 
정렬을 하고 0~1000까지 다 비교하는거
인덱스 기준을로 만약 50 이라고 하면 그 정렬값이 50보다 크거ㅓ나 같으면 나머지도 다 큰거니 문제 없을듯
오름 차순 정렬로 풀이
