## 제한조건
별도로 시간제한, 메모리 제한 조건이 없음... 물어봐야할듯..

## 시간 복잡도 분석 및 전략
O(N) -> 30번의 계산으로 처리가능


### 해결방안
Hash 자료구조를 사용 

### 예외케이스


### 고민했던 흔적들
30개의 배열이 들어오며서 모든 경우의수를 파악해야함
선형적으로 한다고 했을때 코니가 옷을 입는다 안입는다에  대한 케이스를 고려해야함
만약 예를들어 안경, 모자 를 입는다 안입는다 두개의 케잉스로 한다면 경우의 수는 4개가 됨
만약 이를 통해 30개의 의상ㅇ에 적용하게 되면 O(2^N) = 10억 정도의 케이스로 시간 초과 발생
조합으이 경우로 단순하게 시가능ㄹ 계산해보려함 ㅣ를 통해 안입는 경우의 수를 각 카테고리 별로 추가하여
경우의수를 도출하고 모든경우의 수에서 아무것도 안입는다는 케이스는 없으므로 최종적으로 -1 을 하여 답을ㄹ 도출
