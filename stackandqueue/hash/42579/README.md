#문제 풀이
링크 
프로그래머스 베스트앨범 (링크 : https://school.programmers.co.kr/learn/courses/30/lessons/42579)

## 제한조건
genres[i]는 고유번호가 i인 노래의 장르입니다.
plays[i]는 고유번호가 i인 노래가 재생된 횟수입니다.
genres와 plays의 길이는 같으며, 이는 1 이상 10,000 이하입니다.
장르 종류는 100개 미만입니다.
장르에 속한 곡이 하나라면, 하나의 곡만 선택합니다.
모든 장르는 재생된 횟수가 다릅니다.

## 시간 복잡도 분석 및 전략
최초 입력 시점 O(N) N에 데이터를 큐에 삽입할떄 큐에 삽입하므로 O(NlogN)의 시간복잡도
및 결과 정렬시 O(GlogG)     
두개의 시간복잡도를 가질때 더 큰값을 기준으로하면
O(NlogN)의 시간복잡도를 가지게됨

### 해결방안
Hash 자료구조를 사용 
정렬당시 바로 비교 후 데이터를 넣는 방식으로 구현 추후 마지막 정려 작업

### 예외케이스
장르에 속한 곡이 하나라면, 하나의 곡만 선택합니다.

### 고민했던 흔적들
10000개의 입력값이 들어오며 이를 기준으로 장르와 플레이 된 횟수를 기준으로 삼아
각 장르별 가장 많이 재생된 플레이수를 최대 2개를 배열순서에 맞게 출ㄹ력합니다.

풀이를 할때 각 장를별로 해시를 만드는데 이중 햇시가 필요 
Hash<장르, hash<0, 장르길이>
Hash<장르, hash<1, 장르길이>

이러식으로 그래서 새로운 장르가 추가 될대마다 기존 장르에서 2개의 비교 케이스만 추가되면 되지않을까?

O(N*2) 이렇게? 마지막으로 정렬하는거까지하면
O(N*2 + NlogN) 이렇게? 
그러면 이중 순회하는거에 비해서 훨씬빨리 끝날꺼같은데
