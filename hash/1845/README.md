### 고민했던 흔적들
입력값이 1만인 배열과 20만인 케이스가 입력으로 들어옴 선형으로 분석하게 되면 
1만*20만이라서 20억이 되 시간초과가 발생될 가능성 있음
단순 해쉬로 처리한다고 했을ㄹ대 1만에대한 입력값만 입력받고 배열 사이즈보다 해쉬 사이즈가 작으면
그만큼 경우의 수가 작다는것이니 해당 케이스가 정답

정렬 후 풀이하는것도 있을것 같으나 O(NlogN)으로 시간복잡도는 해쉬가 좀 더 유리 공간복잡도에서는
정렬이 좀더유리?

    시간복잡도 공간복잡도
해시 O(N)        O(N)
정렬 O(NlogN)    O(1)
